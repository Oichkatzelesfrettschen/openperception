## Executive Summary

Neurodivergence encompasses natural variations in human cognition, including conditions such as autism, ADHD, dyslexia, and dyscalculia. Current research estimates that 15-20% of the population is neurodivergent, with some studies suggesting up to 40%. This report synthesizes recent academic research (2020-2025) on designing accessible digital interfaces for neurodivergent users.

Primary source lane:

- [Neurodivergence source cache](/home/eirikr/Github/openperception/docs/external_sources/neurodivergence_source_cache.md)
- [Neurodivergence primary source notes](/home/eirikr/Github/openperception/research/neurodivergence/primary_source_notes.md)

### Key Findings

- **ADHD**: Interface design should prioritize simplicity, progressive disclosure, and distraction-free environments
- **Autism**: Sensory-friendly design requires predictable layouts, reduced animations, and customizable sensory input
- **Dyslexia**: Specialized fonts show mixed results; general typography principles (sans-serif, adequate spacing) prove more reliable
- **Dyscalculia**: Numerical interfaces benefit from visual representations, simplified displays, and rounded numbers
- **Cognitive Load**: Minimizing extraneous cognitive load through clean design benefits all users

## ADHD and Interface Design

### Academic Papers

#### 1. HI Applications for ADHD Children: The Graphic Design Perspective (2024)
**Source**: [ResearchGate](https://www.researchgate.net/publication/378813433_HI_Applications_for_ADHD_Children_The_Graphic_Design_Perspective)

ADHD affects approximately 2-8% of children globally. This study addresses digital teaching-learning devices focusing on curriculum-based digital content for enhancing attention spans.

**Key Findings**:
- Optimal font types, styles, line spacing, and background colors substantially improve attention
- Illustration style has marginal effect compared to typography
- Visual design elements (VDEs) can be tailored for curriculum-based digital content

#### 2. Towards Inclusive Guidelines for Web Design for Adults with ADHD
**Source**: [Springer INTERACT 2025](https://link.springer.com/chapter/10.1007/978-3-032-05008-3_59)
**Authors**: Weyerhauser & Piccolo

A comparative evaluation with 14 participants (half diagnosed with ADHD) using an online banking website and a redesigned ADHD-friendly version.

**Key Findings**:
- Participants with ADHD experienced greater challenges with original website
- Both groups benefited from redesigned version
- Promising direction for developing ADHD-specific design guidelines

#### 3. Accessible Design in IDEs for Students with ADHD
**Source**: [Springer](https://link.springer.com/chapter/10.1007/978-3-032-03870-8_15)

Think-aloud study with nine university computing students analyzing Visual Studio Code IDE usage.

**Key Findings**:
- Identified key areas of consideration for IDE design
- Highlighted frustration and barriers in current IDE layouts
- Need for ADHD-aware development environments

#### 4. Interaction Design Strategies for ADHD Learning Attention (2025)
**Source**: [Springer ICITA 2024](https://link.springer.com/chapter/10.1007/978-981-96-1758-6_27)

Literature review on attention problems due to malfunctioning executive function and working memory.

**Key Findings**:
- Need for user-centric solutions for ADHD attention improvement
- Incorporation of ML and AI-based interfaces recommended
- Executive function deficits require specialized design approaches

#### 5. AI Framework for ADHD-Affected Professionals
**Source**: [arXiv 2025](https://arxiv.org/html/2507.06864v1)

Addresses "task paralysis," fatigue, and prioritization confusion in productivity tools.

**Key Findings**:
- Conventional productivity apps offer limited adaptability to fluctuating cognitive states
- Participatory design methodologies emerging from cognitive accessibility communities
- Emphasis on designing with, not just for, neurodivergent users

### ADHD Design Principles

Based on research, ADHD-friendly interface design should include:

1. **Simplify interfaces** - Reduce cognitive load with clear hierarchy
2. **Progressive disclosure** - Show one step at a time (e.g., "Step 2 of 4")
3. **Typography** - High-readability sans serifs, great line spacing, moderate contrast
4. **Reduce distractions** - Avoid auto-playing content, excessive moving parts
5. **Control over motion and sound** - User-controllable animations
6. **Structured, calm interfaces** - Help with focus and task completion

### ADHD Task Management App Research

**Source**: [Tiimo](https://www.tiimoapp.com/) - iPhone App of the Year 2025

Effective ADHD app features:
- Visual timelines to combat "time blindness"
- AI-powered task breakdown for executive function support
- Minimal steps between opening and using
- Customizable reminders, colors, and categories
- Low-distraction, uncluttered layouts
- Gamified elements for motivation

## Autism Spectrum and Sensory-Friendly Design

### Academic Papers

#### 1. Designing Sensory Adaptive Environments for Autistic Children in Healthcare (2024)
**Source**: [Springer Discover Psychology](https://link.springer.com/article/10.1007/s44202-024-00118-5)

Three main solutions emerged:
- Sensory Adaptive Environment (SAE)
- Sensory adaptations during medical procedures
- Provider training on sensory features in autism

#### 2. Developing a Taxonomy for Sensory-Informed Architectural Design (2024)
**Source**: [ResearchGate](https://www.researchgate.net/publication/388898596_Developing_a_taxonomy_for_sensory-informed_architectural_design_qualities_in_autism)

Introduces a taxonomy classifying sensory and spatial design qualities, identifying 83 sensory-informed design qualities through review of 76 sources.

**Critical Gaps Identified**:
- Reliance on intuition for sensory zoning
- Emphasis on interior design over spatial configuration
- Need for quantifying sensory drivers

#### 3. Guidelines of Inclusive Architecture Design for ASD (2024)
**Source**: [PMC](https://pmc.ncbi.nlm.nih.gov/articles/PMC11860188/)

Comprehensive review of architectural design guidelines for autism spectrum disorder.

#### 4. ASPECTSS Design Framework
**Source**: [Magda Mostafa, 2013 - Updated Research](https://www.researchgate.net/publication/378233656_The_role_of_architectural_and_interior_design_in_creating_an_autism-friendly_environment)

Seven design concepts:
1. **Acoustics** - Sound management
2. **Spatial Sequencing** - Logical flow
3. **Escape Space** - Quiet retreat areas
4. **Compartmentalization** - Defined zones
5. **Transition Spaces** - Gradual environment changes
6. **Sensory Zoning** - High/moderate/low stimulation areas
7. **Safety** - Secure environments

### Digital Interface Design for Autism

**Source**: [Accessibility.com](https://www.accessibility.com/blog/sensory-friendly-design-creating-digital-spaces-that-support-autistic-users)

#### Elements to Avoid
- Auto-playing videos
- Flashing carousels
- Incessant notifications
- Excessive animations
- Pop-up ads
- Busy backgrounds

#### Design Recommendations

1. **Visual Design**
   - Clean, uncluttered layouts
   - Limited color palette
   - Soft, muted colors from nature (biophilic design)

2. **Navigation**
   - Clear and intuitive
   - Consistent design patterns
   - Minimal complex animations

3. **Multimedia**
   - Controls to adjust/mute sound
   - No auto-playing audio/video
   - Closed captions and transcripts

4. **Customization**
   - Adjustable color schemes
   - Controllable audio levels
   - Less visually intense interface options

### Prevalence Data

- CDC: ~1 in 36 children in US diagnosed with autism
- WHO: ~1 in 100 people worldwide are autistic

## Dyslexia and Typography Research

### Font Effectiveness Research

#### OpenDyslexic Font Studies

**Source**: [PMC 2017 Study](https://pmc.ncbi.nlm.nih.gov/articles/PMC5629233/)

Mixed results from controlled studies:

**Studies Showing No Benefit**:
- OpenDyslexic reduced reading speed and accuracy compared to Arial and Times New Roman
- Students did not prefer OpenDyslexic
- Dyslexie font showed no benefit to reading accuracy and speed

**Studies Showing Some Benefit**:
- One study found OpenDyslexic improved reading comprehension (but not speed)
- 2023 study showed effectiveness in increasing reading fluency for some students

#### 2024 Copenhagen Research
**Source**: Aarhus School of Architecture

Confirmed that increasing x-height and lengthening character strokes improves legibility.

### Typography Recommendations

**Source**: [British Dyslexia Association](https://dyslexiefont.com/en/research/)

#### Effective Typography Features
- Sans-serif fonts (Arial, Verdana, Tahoma)
- Monospaced fonts where appropriate
- Roman (non-italic) styles
- Font size 12-14 points (minimum 16px on screen)
- Adequate line spacing

#### Features That Hinder Reading
- Serif fonts
- Italic/oblique fonts
- Inadequate inter-letter spacing without increased word spacing
- Very high or very low contrast

#### Background Colors
Research suggests yellow, peach, and orange provide readable backgrounds.

### Bionic Reading

**Source**: [Healthline Analysis](https://www.healthline.com/health/adhd/social-bionic-reading-for-adhd)

A method that bolds the first few letters of words to create "fixation points."

**Research Status**:
- Limited scientific evidence (only 12 participants in original study)
- 2024 Utrecht University EEG study conducted
- Results remain inconclusive
- May work for some individuals but not validated at population level

## Dyscalculia and Numerical Interface Design

### Prevalence

- ~1 in 20 people have dyscalculia in UK
- Low numeracy affects ~17 million working-age adults in UK

### Research and Design Guidelines

**Source**: [UK Government Design Notes](https://designnotes.blog.gov.uk/2022/11/28/designing-for-people-with-dyscalculia-and-low-numeracy/)

#### Design Recommendations

1. **Remove unnecessary numbers** - Round to whole numbers
2. **Eliminate decimal places** - Unless for money
3. **Leave lots of space** - Prevents numbers "jumping around"
4. **Involve users** - Include people who struggle with numbers in design
5. **Use visual representations** - Timelines, progress indicators
6. **Simplify interfaces** - Reduce cognitive load for numerical tasks

### Assistive Technology

#### EDSense
**Source**: [PMC](https://pmc.ncbi.nlm.nih.gov/articles/PMC12109858/)

Early Detection and Intervention for Insufficient Number Sense - adaptive web-based learning tool.

**Features**:
- Personalized support
- Targets skill refinement
- User-friendly interface addressing mouse difficulties
- Designed for dyscalculic learners

#### Calcularis 2.0
**Source**: [PMC](https://www.ncbi.nlm.nih.gov/pmc/articles/PMC7373797/)

Adaptive training program based on neuro-cognitive theory of numerical cognition.

**Features**:
- Automatizes number representations
- Supports mental number line formation
- Trains arithmetic operations and fact knowledge
- 42 sessions of 20 minutes over 13 weeks

### Core Deficits Research

**Source**: [Loughborough University CMC](https://blog.lboro.ac.uk/cmc/2025/07/17/core-numerical-deficits-in-severe-developmental-dyscalculia/)

- Children with severe dyscalculia show difficulties in both symbolic and non-symbolic numerical processing
- Slower and more errors in digit tasks, number sequences, dot comparisons
- Specific weaknesses in how numerical information is processed (not broader learning issues)
- Deficits do not resolve with age

## Cognitive Processing and UX

### Cognitive Load Theory in UX

**Source**: [Interaction Design Foundation](https://www.interaction-design.org/literature/topics/cognitive-load)

#### Types of Cognitive Load

1. **Intrinsic** - Inherent complexity of the task
2. **Extraneous** - Mental processing from distracting design elements
3. **Germane** - Processing that helps learning/understanding

#### Design Strategies

**Source**: [Nielsen Norman Group](https://www.nngroup.com/articles/minimize-cognitive-load/)

1. **Avoid visual clutter** - Remove redundant links, irrelevant images
2. **Build on mental models** - Use familiar patterns and layouts
3. **Group related items** - Avoid split attention effect
4. **Use white space** - Improves readability and visual hierarchy
5. **Progressive disclosure** - Reveal information gradually

### The Psychology of Attention (November 2024)

**Source**: [UX Matters](https://www.uxmatters.com/mt/archives/2024/11/the-psychology-of-attention-for-ux-designers.php)

**Bottleneck Theory**: When multiple stimuli compete for attention, the brain can focus on only one at a time while filtering others.

**Design Implications**:
- Minimize competing attention demands
- Create clear focus points
- Reduce notification overload

### Processing Speed Considerations

**Source**: [MDN Cognitive Accessibility](https://developer.mozilla.org/en-US/docs/Web/Accessibility/Guides/Cognitive_accessibility)

#### Time Limit Accommodations

- Allow users to turn off or adjust time limits
- Provide at least 10x the original limit
- Alert users with 20-second buffer to extend timeout
- Consider eliminating time constraints entirely

#### Barriers to Information Access
- Attention deficits
- Processing speed variations
- Short-term memory limitations

## W3C COGA Guidelines

### Overview

**Source**: [W3C WAI COGA](https://www.w3.org/WAI/cognitive/)

The Cognitive and Learning Disabilities Accessibility Task Force (COGA) develops guidance for making web content accessible to people with cognitive and learning disabilities.

### Key Documents

#### Making Content Usable for People with Cognitive and Learning Disabilities
**URL**: [https://www.w3.org/TR/coga-usable/](https://www.w3.org/TR/coga-usable/)

Published as W3C Working Group Note (April 2021).

**Contents**:
- User stories for cognitive disabilities
- Design patterns with rationale
- How-to guidance and examples
- Test questions for each objective
- Guidance on including users in research

#### Cognitive Accessibility Roadmap and Gap Analysis
Working Draft identifying gaps in current standards.

### 8 COGA Objectives

The document provides eight objectives with user stories and design patterns:

1. Help users understand what things are and how to use them
2. Help users find what they need
3. Use clear and understandable content
4. Help users avoid mistakes and know how to correct them
5. Help users focus
6. Ensure processes do not rely on memory
7. Provide help and support
8. Support adaptation and personalization

### Key Recommendations from COGA

1. **Use familiar patterns** - Standard conventions (e.g., underlined blue links)
2. **Easy navigation** - Clear layout with visual cues and icons
3. **Clear headings and regions** - Help understand page design
4. **Involve users** - Include people with cognitive disabilities in design process

## WCAG 2.2 Cognitive Accessibility

### Overview

**Source**: [W3C WAI WCAG 2.2](https://www.w3.org/WAI/standards-guidelines/wcag/new-in-22/)

WCAG 2.2 published October 5, 2023, adds 9 new success criteria focused on usability for keyboard navigation, touch targets, cognitive accessibility, and authentication.

### New Cognitive-Related Success Criteria

#### Level A

**3.3.7 Redundant Entry**
Information previously entered must be auto-populated or available for selection.
- Addresses memory difficulties
- Reduces repetitive data entry burden

**3.2.6 Consistent Help**
Help mechanisms must occur in the same relative order across pages.
- Supports users who need predictable help locations
- Reduces cognitive load for finding assistance

#### Level AA

**3.3.8 Accessible Authentication (Minimum)**
Cognitive function tests cannot be required for authentication, with exceptions for:
- Object recognition tests
- Personal content recognition
- Alternative authentication methods

**2.4.11 Focus Not Obscured (Minimum)**
Focused elements must not be entirely hidden by author-created content.
- Benefits keyboard users
- Helps users with cognitive/memory impairments identify focus position

#### Level AAA

**3.3.9 Accessible Authentication (Enhanced)**
Stricter requirements - cognitive tests only allowed with:
- Alternative method available
- Mechanism to assist with test
- No exceptions for object/personal recognition

**2.4.12 Focus Not Obscured (Enhanced)**
Stricter focus visibility requirements.

### AAA Criteria Helping Cognitive Impairments

**Source**: [DWP Digital](https://dwpdigital.blog.gov.uk/2022/06/30/designing-accessible-services-dont-exclude-the-neurodiverse/)

Often overlooked but important:
- **1.4.7 Low or No Background Audio** - Helps auditory processing issues
- **2.2.4 Interruptions** - Helps easily distracted users
- **2.3.2 Three Flashes** - Prevents photosensitive seizures
- **2.3.3 Animation from Interactions** - Helps vestibular disorders
- **2.4.8 Location** - Helps short attention span/memory issues

## Testing Methodologies

### Challenges in Neurodivergent User Research

**Source**: [Springer](https://link.springer.com/chapter/10.1007/978-3-030-25629-6_42)

Traditional HCI methods were created for and validated with able-bodied users. User-centered design methods are not always well-suited for neurodiverse individuals.

### Best Practices

**Source**: [AbilityNet](https://abilitynet.org.uk/news-blogs/3-tips-designing-user-research-neurodivergent-testers)

#### 1. Adapt Survey Methods
- Likert scales pose challenges for neurodivergent individuals
- Ambiguity in numerical meanings creates difficulty
- Consider alternative response formats

#### 2. Address Communication Differences
- Alexithymia (difficulty articulating emotions) is prevalent
- Provide alternative mediums: art, descriptions, mental images
- Avoid limiting emotional expression to predefined options
- Allow sufficient response time

#### 3. Design Inclusive Research
- Conduct research specifically with neurodivergent participants
- Collaborate with accessibility experts and advocacy groups
- Incorporate assistive technology compatibility testing

### Co-Design Approaches

**Source**: [Microsoft Research](https://www.microsoft.com/en-us/research/group/customer-insights-research/articles/designing-for-neurodivergent-students-what-weve-learned-so-far/)

Methods:
- Surveys and interviews with neurodivergent users
- Usability testing with representative participants
- Workshops and design sprints
- Feedback sessions
- Iterative involvement throughout design process

### Key Design Principles from Testing

**Source**: [DevQube](https://devqube.com/neurodiversity-in-ux/)

1. **Flexibility first** - Multiple interaction methods
2. **Simple, literal language** - Avoid idioms and jargon
3. **Predictable navigation** - Persistent UI elements
4. **Multiple task completion paths** - Voice, text, click
5. **Error forgiveness** - Clear recovery options
6. **Manageable steps** - Break complex flows
7. **Progressive disclosure** - Prevent overwhelm

## Color, Contrast, and Motion

### Color and Contrast for Neurodivergent Users

**Source**: [Neurodiversity Design System - Colour](https://neurodiversity.design/principles/colour/)

#### Contrast Considerations

Standard accessibility:
- Minimum 4.5:1 ratio (AA)
- Preferred 7+:1 ratio (AAA) for body text

**However**, high contrast (black on white) can increase cognitive fatigue for:
- Oversensitive visual cortex
- Visual-perceptual disorders
- Dyslexia-related difficulties (Irlen Syndrome)

#### Recommendations

1. **Provide color UI modes** - Beyond light/dark
2. **Avoid strong chromatic contrast** - Opposite color wheel positions create shimmering effect
3. **Use biophilic colors** - Nature-derived, calming hues
4. **Soft, muted colors** - Better for sensory processing disorders
5. **Individual variation** - Some need high-stimulation environments

### Motion and Animation Accessibility

**Source**: [W3C WCAG 2.1](https://www.w3.org/WAI/WCAG21/Understanding/animation-from-interactions.html)

#### WCAG SC 2.3.3: Animation from Interactions

Motion animation triggered by interaction must be disableable unless essential.

#### Vestibular Disorder Considerations

**Problematic animations**:
- Parallax effects
- Scaling/panning large objects
- Swooping, zooming motions
- Scrolljacking
- Swiping animations

**Generally safe**:
- Animations under 0.5 seconds
- Fade in/out transitions
- Subtle button animations

#### CSS `prefers-reduced-motion`

**Source**: [MDN](https://developer.mozilla.org/en-US/docs/Web/CSS/@media/prefers-reduced-motion)

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

#### Best Practices

1. Avoid unnecessary animation
2. Provide user controls to disable animations
3. Respect operating system reduce motion settings
4. Use pause/stop/hide mechanisms for auto-playing content
5. Disable only obtrusive animations, not all motion

## Appendix: Quick Reference Design Checklist

### For ADHD Users
- [ ] Progressive disclosure (one step at a time)
- [ ] Clear visual hierarchy
- [ ] Minimal distractions
- [ ] User control over animations
- [ ] Visual progress indicators
- [ ] Task breakdown support

### For Autistic Users
- [ ] Predictable, consistent layouts
- [ ] No auto-playing media
- [ ] Sensory customization options
- [ ] Clear navigation patterns
- [ ] Reduced visual clutter
- [ ] Literal, clear language

### For Dyslexic Users
- [ ] Sans-serif fonts (Arial, Verdana)
- [ ] Adequate line spacing (1.5+)
- [ ] Font size 12-14pt minimum
- [ ] No italic text for body content
- [ ] Good contrast (but not extreme)
- [ ] Option for colored backgrounds

### For Dyscalculic Users
- [ ] Rounded whole numbers
- [ ] Minimal decimal places
- [ ] Visual representations of data
- [ ] Adequate spacing around numbers
- [ ] Clear number formatting
- [ ] Alternative input methods

### Universal Cognitive Accessibility
- [ ] Consistent help location
- [ ] No cognitive function tests for auth
- [ ] Auto-population of previously entered data
- [ ] Adequate time limits (or none)
- [ ] Clear focus indicators
- [ ] Reduced motion option
