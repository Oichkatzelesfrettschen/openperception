# README From Foundations

Date: 2026-03-22

## What `README.md` really is

`README.md` is the onboarding document.

Its job is to tell a new reader:

- what the repo is,
- what the main components are,
- how to get started,
- where to look next.

So it necessarily compresses details that the deeper docs expand.

## Step 1. What picture it gives

The README presents OpenPerception as a combined:

- research repository,
- algorithm toolkit,
- design-token system,
- accessibility specification set,
- development toolkit.

That is a fair summary of the tree we inspected.

## Step 2. What it emphasizes

The README emphasizes three concrete anchors:

- DaltonLens implementations,
- token/design system,
- evidence-backed specs.

That is useful because those are the three most actionable entry points for a
new technical reader.

## Step 3. What it leaves out

Because it is an onboarding doc, it does not yet fully surface the newer
experimental lanes added in this pass:

- mauve/burgundy accessible pack,
- red/mahogany atmosphere-first pack,
- foundations walkthrough notes,
- comparison page.

That is normal; README drift is expected after a working session.

## Step 4. How to interpret it safely

Treat the README as:

- accurate high-level orientation,
- not a complete current changelog,
- not the finest-grained source of truth for every sub-lane.

For finer-grained current state, use:

- `docs/current-work-inventory.md`
- `ARCHITECTURE.md`
- `MASTER_INDEX.md`

## Bottom line

`README.md` is the repo's front door. It remains broadly accurate, but the new
experimental palette and foundations notes now live one layer deeper in the
docs and token directories.
