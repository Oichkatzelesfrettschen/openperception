# Font Contract Token From Foundations

Date: 2026-03-22

## What `font-contract.json` really is

`specs/tokens/typography/font-contract.json` is the machine-readable version of
the repo's typography approval criteria.

It turns typographic quality into explicit tests for:

- static metrics,
- disambiguation,
- rendering behavior,
- cognitive rhythm,
- compliance levels,
- and test environments.

So this file is not a font list. It is a font-gating contract.

## Step 1. What it does well

This token spec is strongest where it keeps disambiguation central:

- `Il1`
- `0Oo`
- `rnm`
- punctuation and dash distinctions

That is a very practical and accessibility-relevant way to judge fonts.

## Step 2. Why this matters

Typography often stays subjective in design systems. This file resists that by
making font admission criteria measurable.

That is exactly the kind of move OpenPerception tends to make at its best:

- replace taste-only language with contracts and tests.

## Bottom line

`specs/tokens/typography/font-contract.json` is the repo's font-gating
contract. It is a strong attempt to turn typography approval into a measurable
system rather than a branding preference.
