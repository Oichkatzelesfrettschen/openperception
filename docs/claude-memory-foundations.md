# Claude Memory From Foundations

Date: 2026-03-22

## What `CLAUDE.md` really is

`CLAUDE.md` is the repo's AI-assistant orientation memo.

It gives a compact operational model of:

- what the project is,
- where the major subsystems live,
- how to build and test them,
- and what conventions matter.

So this is not part of the product architecture directly. It is part of the
repo's maintenance and collaboration architecture.

## Step 1. What it does well

The file is useful because it compresses a large repo into a practical memory
aid:

- core algorithm paths,
- specs,
- docs,
- commands,
- common tasks,
- known issues.

That makes it genuinely valuable for onboarding both humans and assistants.

## Step 2. Where it now lags

Because the repo has grown during this pass, the memo is now behind current
state in a few places.

Notably, it does not yet mention:

- the new experimental palette lane,
- the palette comparison example,
- the red-theme triage and foundations lane,
- the large family of new foundations notes.

So it remains useful, but it is no longer a complete operational snapshot.

## Bottom line

`CLAUDE.md` is the repo's AI-assistant orientation memo. It is a helpful
operational summary, but it now has clear update debt relative to the new
experimental and foundations-document work already present in the tree.
